# Unseen — непрочитанные сообщения для GLPI 11

Нативный плагин для GLPI 11.0.x: помогает не пропустить новую активность в заявках.

## Возможности

- 🔔 **Колокольчик в шапке.** Рядом с меню пользователя появляется иконка‑колокольчик
  с числом непрочитанных заявок и выпадающим списком (ссылки на заявки + дата
  последней активности). Обновляется при открытии и периодически.
- ✅ **Кнопка «Отметить всё как прочитанное»** в выпадающем списке колокольчика —
  одним кликом очищает все непрочитанные заявки пользователя.
- 📖 **Автоматическая отметка прочтения при открытии заявки** — выполняется на
  стороне сервера при отрисовке формы (не зависит от JavaScript), поэтому заявка
  гарантированно помечается прочитанной, как только вы её открыли.
- 🔴 **Подсветка списков.** В любых списках/результатах поиска строки **ваших** заявок
  (где вы заявитель/назначенный/наблюдатель) с непрочитанной активностью подсвечиваются
  (красная метка и жирный текст) — то же множество, что и в колокольчике, поэтому кнопка
  «Отметить всё как прочитанное» снимает и подсветку в списке.
- 〽️ **Разделитель «Новые сообщения» в таймлайне.** В ленте заявки рисуется линия
  ровно на границе того, что вы уже читали, — видно, с какого места продолжить.
  Корректно работает и в прямом, и в обратном порядке ленты.
- 🏢 **Настройка по организациям (entity)** с тремя режимами и наследованием.

## Режимы

| Режим | Поведение |
|-------|-----------|
| **Отключено** | Плагин неактивен в этой организации. |
| **Индивидуально на портале; общее для техников** | Заявители (упрощённый интерфейс) имеют личную очередь; техники (стандартный интерфейс) делят общую — если один техник отметил заявку прочитанной, она прочитана для всех техников. |
| **Индивидуально для каждого пользователя** | У каждого пользователя своя очередь непрочитанного. |

Режим задаётся **по организации** с возможностью наследования от родительской.
Глобальное значение по умолчанию задаётся отдельно.

## Что считается «новой активностью»

Заявка считается непрочитанной, если есть активность от **другого** пользователя
(не от вас), более свежая, чем момент, когда вы в последний раз открывали заявку:

- новые комментарии (followups),
- новые задачи (tasks),
- решения (solutions),
- сама заявка, созданная другим пользователем (чтобы новые заявки тоже подсвечивались).

Приватные комментарии/задачи не учитываются для пользователей упрощённого интерфейса.

## Требования

- GLPI **11.0.x**
- PHP **8.1+**

## Установка

1. Скопируйте каталог плагина в каталог плагинов вашего GLPI так, чтобы путь был
   `.../glpi/plugins/unseen` (**имя папки обязательно `unseen`** — от него зависит
   автозагрузка классов).
2. В GLPI откройте **Настройки → Плагины** (Setup → Plugins).
3. Напротив «Unseen messages» нажмите **Установить**, затем **Активировать**.

После активации в корневой организации включается режим «Индивидуально для каждого
пользователя» — плагин сразу работает.

## Настройка

- **Глобально:** Настройки → Основные (Setup → General) → вкладка **Unseen messages** —
  режим по умолчанию.
- **По организациям:** Администрирование → Организации (Administration → Entities) →
  выбрать организацию → вкладка **Unseen messages** — режим для организации
  (с опцией наследования).

## Как это устроено (технически)

- Точки интеграции — штатные хуки GLPI:
  - `add_css` / `add_javascript` — внедрение ассетов на все авторизованные страницы;
  - `pre_show_item` — отрисовка разделителя «Новые сообщения» в таймлайне;
  - `pre_item_form` — отметка заявки прочитанной при показе её формы;
  - `item_purge` — очистка статусов при удалении заявки.
- Колокольчик и подсветка списков добавляются на стороне клиента (JS) и обращаются
  к AJAX‑эндпоинтам плагина. Отметка о прочтении делается **на сервере** при показе
  формы заявки (через `register_shutdown_function`, после захвата прежней позиции —
  чтобы разделитель в таймлайне остался корректным).
- CSRF для POST‑запросов проверяется штатным механизмом ядра GLPI 11
  (заголовок `X-Glpi-Csrf-Token`), отдельной настройки не требуется.

### Таблицы БД

- `glpi_plugin_unseen_readstatuses` — отметки прочтения `(itemtype, items_id, users_id, last_read)`
  (`users_id = 0` — общая отметка для режима «общее для техников»).
- `glpi_plugin_unseen_entityconfigs` — режим по организациям `(entities_id, mode)`.

## Удаление

В **Настройки → Плагины** нажмите «Деинсталлировать» — плагин удалит свои таблицы и
настройки.

## Структура файлов

```
unseen/
├─ setup.php                     # инициализация, версия, регистрация хуков
├─ hook.php                      # install/uninstall (создание/удаление таблиц)
├─ src/
│  ├─ Config.php                 # глобальная настройка (режим по умолчанию)
│  ├─ EntityConfig.php           # вкладка и хранение настройки по организациям + наследование
│  ├─ ReadStatus.php             # ядро: отметки прочтения и вычисление непрочитанного
│  └─ Timeline.php               # разделитель «Новые сообщения» + очистка при purge
├─ ajax/
│  ├─ menu.php                   # данные для колокольчика
│  ├─ status.php                 # статус непрочитанного для подсветки списков
│  └─ markallread.php            # «отметить всё как прочитанное»
├─ front/
│  ├─ config.php                 # глобальная страница настроек
│  └─ entityconfig.form.php      # обработчик формы настройки по организации
├─ public/                       # статика (GLPI 11 отдаёт ресурсы плагина из public/)
│  ├─ css/unseen.css
│  └─ js/unseen.js
└─ locales/                      # en_GB, ru_RU + шаблон unseen.pot
```
